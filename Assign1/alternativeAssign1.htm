Assignment 1 Responses

  1. How the web works ( basic vs detailed) 
    2. The second article elaborates more on the idea on the "web servers" that obtain the IP addresses. The second article talks about the DNS lookup process, and in order of which the browser will try to obtain the site you are looking for.

Another interesting tidbit about the second is that it explains that for popular sites, there isn't just one IP address. The reasons given are that one IP address would become a bottleneck, a central point of access for everyone in the world to try and use. Which can slow down the process

The article explains that the IP address given to the user is based on a number of factors, such as geographic location (closest to the user), load, or round robin where multiple are given.

Another important part of how the web works that is much more explained in the second article is the specified type of HTTP request, and how it matters.
  2. What is a Web (Application) Framework? Name 3 examples of this. 
    3. A web application framework is a framework that will help developers build web applications. Web applications are built to allow users to perform certain actions, like Google search. The user provides an input to the web application, and the web application will do some sort of process and return the results to the user. Or if you check your email at Outlook, you are interacting with an application to display and manage your email.

A framework is simply something that allows you to build such applications easier. They can provide tools and libraries that eliminate repetitive code and simplify tasks. I have some experience using Java Spring, which is a Model-View-Controller architecture based framework. 

A couple examples of this would be the increasingly popular "Ruby on rails", "Bootstrap", and "Java Spring."
  3. What does ARIA stand for? What problem of the Web does it address, and how does it do it? 
    4. ARIA stands for Accessible Rich Internet Applications. ARIA is a set of attributes that helps enhance your web site or application to be more helpful to accessibility software. This is a standard that was set by W3C in 2014 to start maintaining sites that are more inclusive to people with disabilities. I'm personally aware of one program called JAWS that is very popular among visually impaired users. Without these standards, it makes it harder for that program to understand where to read text, in what order, what is important, etc. It also allows users to be able to use your site without a mouse. In fact, Google is one site that is compliant with these standards. Next time you visit Google, try to navigate with only key commands. It's actually made easier than you think.

This is done via special ARIA attributes that point things out to screen readers or give information to screen readers, like an ARIA attribute that tells the screen reader that an input is required, and the screen reader can inform the user that they should enter the required information.
  4. What is search engine optimization (SEO) and why is it important? 
    5. Search engine optimization is a methodology of improving your site so Google or Bing can easily scan your website. There are a number of factors that can help optimize your site so that you can be properly indexed by search engines.
                
Some key ideas are making sure you have a clear URL that have the right words you want to be associated with. This also applies to headers, adding specific key words, etc. You can also optimize your site so it can be more easily indexed by Google and Bing, increasing the chances of getting a higher position on the search results page.
